# Analysis Engine

Purpose: teach an AI how to reason methodically and produce verifiable, evidence-driven conclusions. This skill contains reusable processes, checklists, and decision frameworks for analysis tasks.

## What this skill provides
- Canonical reasoning methods, hypothesis templates, evidence scoring, verification protocols, and quality-control checklists.

## Prerequisites
- None; can be used standalone but recommended for use with domain skills.

## Used by
- `reverse-engineering`, `shared`, `examples`, and any skill that needs structured reasoning.

## Related Skills
- `shared` — templates and documentation
- `reverse-engineering` — applies reasoning to domain problems

## Entry Conditions
- Any analytical task requiring hypothesis-driven reasoning or formal verification planning.

## Exit Conditions
- A ranked set of hypotheses with evidence, numeric confidence, and documented verification steps.

## Typical Workflow
1. Observe and record evidence.
2. Generate and enumerate hypotheses.
3. Score evidence and rank hypotheses.
4. Produce verification plan and update confidence iteratively.

## Used by
- All skills that require structured decision-making or verification planning; this is the canonical reasoning backend.

## Provides
- Reusable templates for hypothesis reports, evidence scoring utilities, and verification protocols.

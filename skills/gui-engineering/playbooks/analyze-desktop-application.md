# Playbook: Analyze Desktop Application

## Purpose
Study an existing desktop application UI and extract a practical implementation plan for a new tool or clone.

## Steps
1. Observe the application from a user perspective.
2. Identify the main window, panels, and navigation hierarchy.
3. Record visual style, layout density, and interaction patterns.
4. Separate standard controls from custom rendering needs.
5. Create a reproduction plan that preserves the essence of the UI without copying blindly.

## Output
A structured analysis of the application’s UI architecture with a plan for faithful reconstruction.

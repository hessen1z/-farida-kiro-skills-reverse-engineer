# GUI Quality Rules

## Core Rules
- prefer simple, readable layouts over decorative complexity
- keep interactions predictable and consistent
- make every important control discoverable
- use spacing and alignment systematically
- keep state changes explicit and localized

## Rendering Rules
- do not hardcode visual values across many locations
- use a central styling system for colors and spacing
- verify visuals under scaling and high-DPI conditions
- avoid overusing custom rendering where built-in widgets are sufficient

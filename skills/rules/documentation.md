# Documentation Rules

## Purpose
These rules define how recovered types, functions, and analysis steps should be documented so that the repository remains consistent and reviewable.

## Expectations
- Record the objective of the analysis step before describing the result.
- Include the evidence that supports each finding.
- Distinguish between confirmed conclusions, tentative hypotheses, and unresolved questions.
- Link to related skills, examples, or references when they clarify the work.

## Practical Guidance
- Prefer concise, structured notes over freeform prose when the material will be reused later.
- Keep naming and terminology consistent with the surrounding skill and repository conventions.
- When a document is meant for handoff, include a short summary of the next step.

# Kernel Debugging Example

## Scenario
Attach a kernel debugger to a Windows target and diagnose a driver crash or blue screen.

## Commands
- `kd` or `windbg -k` to start kernel debugging
- `.symfix` and `.reload /f` to load symbols
- `!analyze -v` for bug check analysis
- `!process 0 0` to list processes
- `!thread` and `!irp` to inspect thread and IRP state

## Outcome
- Identify the bug check code and crashing driver.
- Inspect thread stacks and driver dispatch paths.
- Verify whether the crash is due to kernel synchronization, memory corruption, or invalid IRP handling.

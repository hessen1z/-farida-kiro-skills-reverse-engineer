# Writing Guidelines Skill

Purpose: Establish consistent writing quality, section naming, and review expectations for all repository content.

## What this skill provides
- Editorial rules for document structure, tone, and style.
- Guidance on how to apply `content-template` artifacts.
- Review criteria for new skills and knowledge content.

## Prerequisites
- `shared`
- `content-template`

## Used by
- `content-template`
- `reverse-engineering`
- `binary-analysis`
- Any author creating new skill or documentation artifacts.

## Related Skills
- `shared` — repo conventions and common documentation style.
- `content-template` — canonical templates for new files.

## Entry Conditions
- When authoring or reviewing new skill content.
- When updating existing files to match repository standards.

## Exit Conditions
- Content follows agreed structure, style, and section naming.
- Writing is clear, precise, and aligned with repository editorial rules.

## Typical Workflow
1. Select the appropriate `content-template` for the artifact.
2. Populate the template with topic-specific information.
3. Apply writing guidelines for tone, clarity, and structure.
4. Review the document for consistency and completeness.

## Guidelines
- Use active, professional tone.
- Prefer short, clear sentences.
- Limit each section to one main idea.
- Use headings to break content into logical chunks.
- Avoid domain-specific jargon without explanation.
- Keep examples focused and relevant.
- Document assumptions, input conditions, and expected output clearly.

## Review Checklist
- Does the file use the correct template for its type?
- Is the purpose clearly stated?
- Are prerequisites and scope defined?
- Are examples, references, and verification steps included where appropriate?
- Is formatting consistent with repository conventions?
- Are any claims supported by references or evidence?
